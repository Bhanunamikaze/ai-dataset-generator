# data-card

Use this during export finalization.

## Required coverage

- dataset purpose
- task types
- example counts
- split sizes
- source types
- collection method
- quality checks applied
- dedup strategy
- known risks or biases
- license note or placeholder

`scripts/export.py` already writes `workspace/DATA_CARD.md`. Expand or rewrite it if the user wants a richer narrative card.
